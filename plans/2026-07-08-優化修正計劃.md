## 3. 任務詳細條目

### P0-1｜修 Bug：concurrency group 無效 `[done]`

- **背景**：`.github/workflows/daily-fetch.yml` 的 `group: daily-fetch-${{ github.run_id }}` 用了 run_id，每 run 自成一組，併發保護形同虛設 → 兩 run 同時讀改寫 latest.json 會 lost update（融資被蓋掉的偶發問題根源之一）
- **改動檔案**：`.github/workflows/daily-fetch.yml`
- **做法**：`group` 改為固定字串 `daily-fetch`，`cancel-in-progress: false` 保留（後到的排隊不取消）
- **AC**：手動連續觸發兩次 workflow_dispatch，第二個 run 在 Actions UI 顯示 pending 排隊直到第一個結束
- **注意**：同時更新 yml 內註解（原註解「後進的直接取消」描述錯誤）

### P0-2｜修 Bug：buildChipsEntry null crash `[done]`

- **背景**：`scripts/fetch-daily.mjs` 的 `buildChipsEntry()` 中 `pcr: phase1.pcr` 與 `vix: phase1.vix` 少了 `?.`。設計上允許 phase1=null（n8n 未寫入但融資已就緒）建 partial entry，此時直接 TypeError 整個 run 失敗
- **改動檔案**：`scripts/fetch-daily.mjs`（搜尋 `phase1.pcr`）
- **做法**：改為 `phase1?.pcr ?? null`、`phase1?.vix ?? null`
- **AC**：`SUPABASE_URL=... node -e` 單測 `buildChipsEntry(null, 123.45, 120)` 不丟例外且回傳物件 `margin_amount === 123.45`、`pcr === null`

### P0-3｜Guard 前置 + meta.json `[done]`

- **背景**：Guard 1 目前下載整包 latest.json（~10MB）只為看幾個日期戳；且 workflow 先跑 npm ci 才執行 Guard。被攔截的 run 白耗 2 分鐘 + 10MB egress × 每天多班 × 每月 → 配額主要浪費源
- **改動檔案**：`scripts/fetch-daily.mjs`、`scripts/write-firebase.mjs`、`.github/workflows/daily-fetch.yml`
- **做法**：
  1. `write-firebase.mjs` 額外上傳 `meta.json`：`{ updatedAt, stocksDate, twseDate, tpexDate, chipsDate, sectorDate, signalsDate, indexDate, sectorHistoryLen }`（欄位齊備供 P1-6 用）
  2. `fetch-daily.mjs` Guard 1 改讀 meta.json 判斷；meta.json 404 時 fallback 舊邏輯（過渡相容）
  3. yml 調整順序：先跑一個「guard-check」step（純 node，不需 npm ci），輸出 `SHOULD_RUN`；為 false 直接結束；true 才 npm ci + 完整 pipeline
- **AC**：(a) 被攔截的 run 總時長 < 1 分鐘（Actions UI 確認）；(b) meta.json 可公開讀取且欄位齊全；(c) 正常 run 行為與改造前一致（latest.json 內容結構不變）

### P1-3｜泡泡圖單位改億元 `[done]`

- **背景**：目前 X/size 用「張」，低價股權重被高估。Tide 用億元，金額才能跨板塊比較
- **改動檔案**：`scripts/fetch-daily.mjs`（`fetchT86Sectors`）、`lib/calcSectors.ts` + `scripts/calc-sectors.mjs`（兩份同邏輯都要改）、`lib/types.ts`、`components/bubble/BubbleChart.tsx`（軸標籤）、`components/bubble/SectorPanel.tsx`（個股列顯示）
- **做法**：T86 分組時股數 × 該股收盤價（stockMap 有 close）→ 億元（`shares * close / 1e8`）。`SectorDayData.rows` 的 net/buySell 與 stocks[].net 等欄位全部改存億元；**sectorHistory 舊資料是張數，需一次性重建**（用 `seed-sectors.mjs` 重抓 25 天，或寫轉換腳本按當日收盤價換算——建議重抓，準確）
- **AC**：(a) 泡泡圖 X 軸單位顯示「億元/日」；(b) 抽一板塊手算驗證：Σ(個股外資淨買股數×收盤價)/1e8 與顯示值一致（±1%）；(c) SectorPanel 個股列的三大法人欄顯示億元
- **注意**：這會改變 sectorHistory 資料格式 → 加 `unit: "yi"` 欄位標記新格式，calcSectors 遇舊格式（無標記）時不混算

### P1-4｜symlog 軸 `[done]`（依賴 P1-3）

- **背景**：板塊資金量級差百倍，線性軸小板塊全擠原點。見 `對數刻度-Symlog-Scale.html`
- **改動檔案**：`components/bubble/BubbleChart.tsx`（`toSVG()` 與軸刻度）
- **做法**：X 軸 symlog：`f(v) = sign(v) * log10(1 + |v|/linthresh)`，`linthresh` 取 1（億元）；刻度標籤仿 Tide：0/±5/±20/±50/±100/±200/±500。Y 軸（加速指標，範圍小）維持線性。trail 座標同步用新映射
- **AC**：(a) 最大板塊與最小非零板塊同時可見且不重疊在原點；(b) 象限縮放（zoom）功能正常；(c) collision resolution 的位移量顯著下降（目測泡泡位置與軸刻度對得上）

- **改動檔案**：`app/aftermarket/page.tsx`、各區塊 component、`lib/types.ts`（snapshot 加各 date 欄位，pipeline 側在 P0-3 已產出）
